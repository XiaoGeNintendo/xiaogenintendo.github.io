Github被墙，Desktop用不了，就算开了VPN怎么办？

今天TM研究了一天这个破烂问题。

问题本质是git不走系统代理，再加上Github Desktop又不支持手动设置代理，只知道憨憨的调用git命令。解决问题的方法非常简单，就是设置git代理：

在命令行中输入(确保git在目录或path下)：

git config --global https.proxy 127.0.0.1:****
git config --global http.proxy 127.0.0.1:****

嗯，然后就完事了。
