类型的类型

在Java中，变量的类型可以是像int,long这样比较单纯，只保存一种信息的基本类型（暂且这么叫）。

同时也有包含着方法、成员变量的String,Scanner这样的复杂类型（非专业术语）。

更有着数组。

那么，三者有什么区别呢？ 请看表：

Characteristic Basic Type Complex Type Array
Examples int, long, boolean String, BigInteger int[], String[]
Passed in function Pass Value Pass Address Pass Address
Default Value 0/false null null
Stored Value value pointer pointer
Query equal? a==b a.equals(b) Arrays.equals(a,b)
compare? a>b a<b a.compareTo(b) -
Create? int a=3 keyword new keyword new

Java中绝大多数类型是复杂类型，其变量实质包含的是一个指向该对象的指针。

什么是null

null，及空指针，指向虚无的指针。对于null调用任何东西会运行时错误，有如从虚空中取物品，非常好理解。

【RTiFE】能力、凶兽与DLord

Introduction

This is a sequal and expansion of RTiFE

Read that before this

能力

【概】能力

Ability

能力分为3种：

  • 种族能力：由你的种族class确定。比如说，所有的火龙都会喷火。

  • 个体能力：某个生物具体的独有的能力，通常是天生的或者早期觉醒的，也有可能是他人供给的。比如说，Rinko的“附身鸟和龙的能力”

  • 凶兽能力：某个凶兽通过杀戮获得的能力，短期能力。成为执政凶兽后容易忘记。比如Rinko的“拍照使物品静止的能力”

凶兽

【概】凶兽

Beast

凶兽是世界中重要的一部分，（一般来说）代表着黑暗与杀戮。好像Temmie的世界观中也有类似的设定

凶兽的（一般）特性：

  • 有着杀戮的欲望。存在凶兽基因，会在虐待他人时获得快感，这种快感在受害者与自己关系越好时越强烈。

  • 能够通过杀死其他有能力的人复制能力

  • 能够通过杀戮和吞噬生物精华（心脏等）获得凶兽能力。

  • 在成为执政凶兽后非常温顺

【概】半凶兽

Half Beast

  • 半凶兽指有着类似凶兽的能力但是不宣告成为凶兽的生物。比如说早期的Rinko有着强大的个体能力但不是凶兽。

  • 半凶兽不可复制能力，没有凶兽基因，没有凶兽能力。

【概】自由凶兽 extends 凶兽

Free Beast

  • 自由凶兽指 自主 宣告成为凶兽的半凶兽。

  • 可以成为执政凶兽，可复制能力，有凶兽基因，有凶兽能力。

  • 龙族不可以成为自由凶兽

【概】契约凶兽 extends 凶兽

Contract Beast

  • 契约凶兽指利用龙王的力量宣告成为凶兽的半凶兽。

  • 不 可以成为执政凶兽，但可复制能力，有凶兽基因，有凶兽能力。

  • 龙族可以成为契约凶兽

【概】执政凶兽

No English Translation

  • 执政凶兽是凶兽的一种状态。触发于毁灭世界或者击败上一届执政凶兽和龙王之后。

  • 执政凶兽可以控制龙王，作为世界代表。

  • 毁灭世界的凶兽有权力定义新世界规则。

龙王

龙王

D. Lord

种族： 龙

年龄： N/A

能力： N/A

介绍：

  • 龙王一般显人形，不显尾、翼、角。红色火龙。

  • 世界的Administrator。拥有的力量基本上是inf。唯一的制约是受到执政凶兽的控制。

  • 检验凶兽是否有资格成为执政凶兽。

  • 战斗时永远不使用过大的力量。（大概肉体力量的10%）

  • 掌握全部龙族的力量和世界的定义权。所有龙族必须服从龙王的命令。
