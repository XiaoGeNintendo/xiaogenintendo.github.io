「考える問題は簡単ですか」

Programming Detect

You are given a directed graph G with N nodes and M edges. Each node has an operation, which could be “def sth” or “use sth”.

An array of nodes A is called Exceptional iff:

  • for each there is an edge connecting and
  • There exists some string S so that the use operation comes earlier than the first def operation of string S.

For example,def i;use i;def j;use j;def j;use j;use i; meets the second condition.
use i;def i;use i;def i is bad because ths usage of i comes earlier than its declaration.

Your task is to find if the given graph contains an Exceptional path.

example

Input
Graph={1->2,2->3,3->1,2->2}
Operation={“def x”,”def y”,”use x”}
Output
NO

Input
Graph={1->2,1->3,2->3}
Operation={“def x”,”def y”,”use y”}
Output
YES
Explain
When A=[1,3] “y” is used without declaration.

Constriants

Subtask 1(9.999999999999999999999%): the operation of each node could only be “def var” and “use var”.”var” is a constant but not a varible.
Subtask 2(20%): the graph is a directed tree.
Subtask 3(30%): it is made sure that for each i, node i and i+1 are connected.
Subtask 4(20%): the graph is a forest
Subtask 5(20%): N,M<=1000
For all, N,M<=10^5

Problem 1006 Final Exam

一开始MK写了一个根号枚举的做法，然后TLE了，我也不知道他怎么写的，没听懂他的做法所以这一段就跳过了反正就是猜想最优解一定是因子，所以就枚举因子，这样的吧
之后syr发现AC submissions全都是15/31/46ms，觉得应该是直接输出答案，所以尝试了cout << n * m + k << endl和cout << k * m + n << endl
能过就见鬼了。。。
Problem 1006’s solution
设每门科目复习的时间为(a_i) - 1, 那么当a_i确定之后出题人需要选出n - k + 1个a_i出来:a_p1, …, a_p(n-k+1)，同时把m搞成一个n - k + 1份的划分: b_1, …, b_(n-k+1)，使得对于每个i，有a_pi <= b_i
那么显然出题人会选最小的n-k+1个a_i。将a排序，则变成了存在一种方案把m分成b_1, …, b_(n-k+1)的方法使得b_i >= a_i
最优的分法是b_i = a_i，所以这个条件等价于sigma(a_i) = m
而总的cost就是所有a_i的和。我们希望这个尽量小，而前n - k + 1个a_i的和已经确定了，就是希望后面的a_i尽可能小，而这个序列是排序过的，所以后面的每一个a_i最小就是a_(n - k + 1)了
所以转化为a_(n - k + 1)尽可能小，而它的最小值就是ceil(m / (n - k + 1))
所以直接输出就好辣！
然后syr就写了这个题

Hypixel Skyblock 简评&amp;新手指导

Skyblock是Hypixel最近新出的一款模式，尽管刚刚推出，但已非常流行，同时在线人数15k+.

当然，我不敢也没有资格对这个消耗了Hypixel工程师无数心血的游戏指点江山，所以只是说一说自我感受.

游玩经历

刚进游戏，大概的游戏界面就像这样（请忽略我丑陋作怪的皮肤） 整个游戏看起来好像和普通的空岛生存无异，但你以为就这样了嘛？完成前几个新手任务，进入Lobby(大厅),真正的面纱才被揭开 ↑事我，大厅 游戏风格逐渐RPG化，有非常多的NPC&Objective，一步一步引导玩家熟悉地形&玩法。 整体建筑风格偏复古(?).Skyblock的神秘面纱被揭露一角. Noob们可以从Farming,Foresting,Mining,Combat这四条主线中获取自己日后非常重要的钱和Health,defense增益(HP和防御). 大概3~5day的亚子就能把所有的任务完成,然后...

你又回到了你的island!

游戏风格再次大转弯，其实是空岛生存2333，逐渐地，你会体验到Skyblock最具特色的一点——Minion(即机器人)

Minions are a fundamental part of Hypixel SkyBlock. They allow the player to receive resources even while offline. --Hypixel Skyblock Wiki

简单来说，可以在任何时候为你工作，赚取你想要的资源.有了这个，你可以做出你所要的装备、武器。同时，由于他可以生成东西，所以可以用于建塔（狂喜），加快工作效率。然后就是漫长的等待，等待你的武器，等待你的装备，但如果你实在是等不及，你也可以在Auction House(即拍卖会)上购买，当然你也可以拍卖自己的东西

你今天ah了嘛? Auction是全服最毒奶的地方。对于我这种非酋来说，别人轻轻松松卖200K的东西在我这里甚至10k都卖不到，我去买别人东西又贵的一批(((( ~我讨厌括号补全~ 当然这也是沟通~东西方~ 玩家之地，大概每天一半消息都和Auction有关.

大概了解了这些，你已经是一名合格的云Skyblock玩家啦！你可以自行探索其余地方，彩蛋不少哦

推荐Armor顺序:

Lapis Armor(获取:Deep Caverns|Auction House) -> Hardened Diamond Armor (获取:Diamond Minion|Auction House) -> Magma Armor (获取:Magma Minion|Auction House) -> End Armor(获取:The End|Auction House) | Emerald Armor (获取:Emerald Minion|Auction House) | 杂套 (Skeleton Hat&Obsdian Chestplate&Creeper Pant&Spider Boot) -> 任意龙套 (获取:The End|Auction House)

Mining&Combat线难度排序

Coal Mine<Gold Mine<Graveyard<Spider’s Den<Deep Caverns (与前者排名视哪个区域而定) < Blazing Fortress < The End

总评

总体来说是一款非常不错的mc server game!自由度非常高，你甚至可以建塔.玩法多样，你可以死命地怼一条线直至功德圆满. 氪金玩家和普通玩家相差不大.Hypixel几乎重写了一切mc原版的东西,使得一切东西拥有Hypixel的魅力.前期可玩度极高，但后期可玩度有所下降，直接上线收点摸鱼.衷心建议Hypixel添加更多PVP的地方，增加更多的Combat Point(The End和其他地方难度差距极大).

Skyblock is the best!

  1. 并不能根据字典序来确定每个序列的第一个数
  2. 所谓的状压dp根本不用算，因为它等于__builtin_popcount(msk)!

Problem 1005's solution #2' 枚举第一个数，找到每个数的前k个permutation，放一块儿排序 复杂度: O(k * n * n * logn(k * n * n)) ≈ 2e7log

于是就愉快地TLE了。
这时候发现这道题的做法越来越暴力了，我就往暴力的方面多想了想。

Problem 1005's solution #2 对于n <= 8的情况， 由于n! <= 40320，可以处理所有permutation然后排序 对于n > 8的情况， 由于前两个数可以确定之后也能保证剩下来的数的排列个数超过k，那么就将前两个数确定为n和1，这样保证了结果序列字典序最小，再在剩下的里面选第k大的。

15: 09: 29 Problem 1005 Passed
但是后来我发现这个做法其实是假的。为什么呢？因为上述做法中n>8的情况“剩下来的书排列个数超过k”是显然错误的 (7! < 10000)。hack数据也很简单：

1
9 10000

dreamoon的数据好水正确的做法是对于n>=10的情况用上述做法中的n>8的情况做，而对于n=9的情况我们可以用一个priority_queue维护排列，一开始插入每个数开头的最小的，然后弹出k个最小的，每弹出一个就把它的next_permutation插入。当然这是后话了。

Problem 1004

当我正在code 1005的时候，XGN指出1004是“零点分段法”。
我：“零点分段法是什么神奇的算法？？？为什么我从来没听说过？？？XGN是怎么会的？？？”
没有多想，我接着写1005。1005 AC之后，XGN那时也没什么想法了。我想，我也帮不了他吧。
于是我就开始颓了去上了个厕所。路上我猛然想到，”零点分段法”不是一种算法，而就是东方上课讲过的一种做数学题的方法。。。鬼知道我一开始在瞎想些什么。。。
然后就醍醐灌顶了，于是我给XGN口胡了一个做法：

Problem 1004's solution 考虑将零点排序，得到x1, x2, ..., xn。对于一个区间[xi, xi + 1), 若有一个解x'在此区间中，那么带入原方程得Σj |aj*x'+bj| = C => Σ1<=j<=i -(aj*x+bj) + Σi<j<=n (aj*x'+bj) = C 观察到一次函数可合并的性质，我们令A = -Σ1<=j<=i aj + Σi<j<=n aj, B = -Σ1<=j<=i bj + Σi<j<=n bj，原方程化为Ax+B = C，就可以愉快地解辣

XGN code完wa了一次，经过一番细致的检查发现是分数类写得不对，即分子分母同为负时没有消负号。
改完就PE了又改完又CE了然后又PE了然后就
